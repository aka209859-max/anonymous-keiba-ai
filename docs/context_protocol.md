# コンテキスト維持プロトコル

このドキュメントは、長時間セッションでのコンテキスト喪失を防ぐための運用ルールです。

## セッション開始時の必須チェックリスト

- [ ] docs/environment_info.md を読み込んだか
- [ ] docs/roadmap.md で現在のフェーズを確認したか
- [ ] docs/prompts.md で今回実行するプロンプトを確認したか
- [ ] 前回のセッションログ（もしあれば）を確認したか

## 20ターンごとの確認事項

1. 現在のタスクを一文で要約
2. 決定事項をリスト化
3. 未決事項を明確化

## セッション終了時のプロトコル

1. 作業内容を docs/session_log.md に記録
2. 次回のタスクを明示
3. 重要な変更を Git コミット

## コンテキストの腐敗（Context Rot）の兆候

以下の症状が出たら、セッションをリセットすること:

- AIが以前の決定事項を忘れている
- 制約条件（当日オッズ除外等）を無視し始める
- 本来の目的（LightGBM学習）から逸脱する
- 同じ質問を繰り返す

## 緊急時の復旧手順

1. 現在のセッションを終了
2. 新しいセッションを開始
3. 本指示文を再度貼り付け
4. docs/ 配下の全ファイルを再読込
